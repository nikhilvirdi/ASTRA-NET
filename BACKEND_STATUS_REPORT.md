## 3. Issues & Bug History

### Resolved Issues
1. **The "Prisma-Timescale" Conflict:** Prisma attempted to manage hypertables as standard tables. **Fix:** Created a custom initialization script that bypasses Prisma for the initial table partitioning.
2. **Satellite Math Latency:** Initial attempts to run SGP4 math in Node.js caused 3-5 second delays. **Fix:** Offloaded this to a Python microservice which handles the math in <50ms.
3. **API Rate Limiting:** NASA APIs were blocking our service during development. **Fix:** Implemented a Redis-based caching layer and smart polling (polling frequency decreases if data hasn't changed).

### Current "In-Progress" Challenges
1. **Memory Footprint:** The Python service currently uses more RAM than targeted when processing large TLE batches.
2. **Socket Latency:** Slight delay noticed when attempting to stream live data via standard HTTP polling.
